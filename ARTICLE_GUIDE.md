# 記事追加ガイド

コードを変更せず、Markdownファイルと写真の追加だけで記事を公開できます。

## 新しい記事の作り方

1. `src/content/interviews/001.md` をコピーします。
2. `003.md` のように、既存と重ならない番号へ変更します。
3. ファイル先頭の `---` で囲まれたFrontmatter（記事の基本情報）を書き換えます。
4. 2つ目の `---` より下へ本文を書きます。
5. `npm run dev` で確認します。
6. 問題がなければ `draft: false` にし、`npm run build` を実行します。

## Frontmatter各項目

| 項目 | 必須 | 意味・記入例 |
| --- | --- | --- |
| `title` | 必須 | 記事タイトル。`"タイトル"` のように引用符で囲みます。 |
| `slug` | 必須 | URLの末尾。半角英小文字とハイフンを推奨（例：`shop-owner`）。重複不可です。 |
| `description` | 必須 | 一覧や検索結果に出る短い説明。80〜120字が目安です。 |
| `publishedAt` | 必須 | 公開日。`"2026-07-14"` の形式です。 |
| `updatedAt` | 任意 | 更新日。更新がなければ行ごと削除できます。 |
| `category` | 必須 | 表示名（例：`"コミュニティ"`）。 |
| `categorySlug` | 必須 | カテゴリURL用の半角英字（例：`"community"`）。 |
| `guestName` | 必須 | インタビュー相手の名前。 |
| `guestRole` | 必須 | 肩書き・活動。 |
| `guestProfile` | 必須 | プロフィール文。 |
| `author` | 必須 | 執筆者名。 |
| `thumbnail` | 任意 | 一覧用写真のパス。未指定なら自動プレースホルダーになります。 |
| `mainImage` | 任意 | 記事上部の写真のパス。未指定でも崩れません。 |
| `mainImageAlt` | 任意 | 写真の内容を説明する文。写真がある場合は必ず書きます。 |
| `featured` | 必須 | トップの特集候補なら `true`、それ以外は `false`。 |
| `draft` | 必須 | 下書きは `true`、公開は `false`。 |
| `tags` | 必須 | 関連語。行頭に半角スペース2つと `-` を付けます。 |

## 本文の書き方

### 見出し

`## 見出しの言葉` と書きます。`#` 1つはページのタイトル用なので、本文では使いません。

### 質問と回答

質問は太字にします。

```markdown
**—— 最初に、この活動を始めた理由を教えてください。**

ここに回答を書きます。質問と回答の間には空行を入れます。
```

### 引用

特に印象的な言葉の行頭に `>` を付けます。

```markdown
> みんなが安心して帰ってこられる場所にしたいんです。
```

## 写真の追加

1. `public/images/interviews/003/` のように記事番号のフォルダを作ります。
2. 写真を入れます。名前は `main.jpg`、`thumbnail.jpg`、`sub-01.jpg` のように、半角英数字とハイフンを使います。
3. Frontmatterへ次のように書きます。

```yaml
thumbnail: "/images/interviews/003/thumbnail.jpg"
mainImage: "/images/interviews/003/main.jpg"
mainImageAlt: "窓辺で話すインタビュー相手"
```

本文中の写真は次の形式です。

```markdown
![写真の内容を説明する文](/images/interviews/003/sub-01.jpg)
```

写真は横幅1600〜2400px程度のJPEGまたはWebPを推奨します。公開前にファイル容量をできれば1MB以下へ圧縮してください。

## カテゴリを追加する

記事の `category` と `categorySlug` に新しい値を書くだけで、カテゴリページが自動生成されます。同じカテゴリの記事では、表記とslugを完全に統一してください。

## 下書き・公開日の変更

- 下書き：`draft: true`（公開サイトの一覧、RSS、サイトマップに出ません）
- 公開：`draft: false`
- 公開日の変更：`publishedAt` を変更すると、一覧の順番も自動で変わります。

下書きは通常の開発画面にも表示されません。確認するときは一時的に `draft: false` にし、確認後に必ず戻してください。
